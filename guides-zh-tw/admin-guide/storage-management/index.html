    提供儲存空間給使用者或群組（Storage Management）¶

    平台上所有持久化資料都存在 Kubernetes PVC（Persistent Volume Claim）。PVC 必須由 Admin 提供 / 建立，使用者無法自行建立。

    權限： 僅 Admin 可建立 / 擴充 / 刪除 PVC。 路徑：Admin Panel → 儲存（/admin/storage）。

    儲存生命週期¶

    flowchart TD
    AdminInit["Admin: 批次初始化\n使用者個人儲存"] --> PVCCreate["建立 K8s PVC\n（每位使用者一份）"]
    PVCCreate --> Available["在儲存頁出現"]
    Available --> PermSet["Admin 設定群組權限"]
    PermSet --> Inherited["專案繼承群組權限"]
    Available --> Expand["擴充 PVC 容量"]
    Available --> Delete["刪除 PVC\n（資料永遠遺失）"]

    個人儲存（User PVC）¶

    每位使用者最多一份個人 PVC。

    批次初始化使用者儲存¶

    1. 進入 Admin → Storage → Users 頁籤。
    2. 點 批次初始化使用者儲存。
    3. 選擇要初始化的使用者（或全選）。
    4. 設定 預設容量（例：50Gi）。
    5. 點 Initialize。

    平台會為每位被選使用者各建立一份 PVC，使用環境變數 DEFAULT_STORAGE_CLASS_NAME 中設定的 storage class（預設 longhorn）。

    截圖建議： /admin/storage Users 頁籤，顯示使用者列表、容量、狀態 badge、批次初始化按鈕。

    檢查狀態¶

    點 批次狀態檢查（Batch Status Check）：

    狀態 意義
    Provisioned PVC 已建立並 bound 到 PV
    Pending PVC 已建立但 storage class 仍在配置中
    Not Initialised 該使用者尚未取得 PVC

    群組儲存（Group PVC）¶

    群組 PVC 是「共享儲存」，整個群組的成員都可以掛載（依個別權限決定可讀 / 可寫）。

    Storage Class 政策¶

    平台透過 GROUP_STORAGE_CLASS_OPTIONS 提供四條儲存路線：

    Storage Class 限制 最佳用途
    longhorn 不限制 預設 personal 與小型 project PVC
    juicefs-gpu23-rwx 限制 跨節點訓練資料 / checkpoint（gpu1/gpu2/gpu3 共享）
    longhorn-gpu23-group 限制 GPU23 /data2 舊版 Longhorn 群組儲存
    longhorn-gpu23-rwx-25g 限制 舊 RWX/NFS 相容

    群組儲存策略：在群組設定的「Storage Policy」會限制此群組能用的 Storage Class，這裡的選單也會自動只顯示該群組允許的選項。

    Fast-stage 政策：longhorn-gpu23-fast-rwo 只保留給專案內的 fast-stage 快取 PVC，不應直接建立成群組 PVC。fast-stage 的來源必須是共享 RWX 群組儲存，例如 juicefs-gpu23-rwx 或明確設定的舊版 RWX fallback。

    建立群組 PVC¶

    1. 切到 Groups 頁籤 → 點 + 新增 Group Storage。
    2. 填寫：
    欄位 說明
    Group 此 PVC 屬於哪個群組
    Name PVC 顯示名稱
    Capacity 容量（例：200Gi）
    Storage Class 視該群組允許的選項
    Description 用途
    1. 點 建立。

    擴充 PVC¶

    1. 在儲存列表找到 PVC。
    2. 點 Expand。
    3. 輸入比目前更大的容量。
    4. 確認。

    PVC 只能擴大，不能縮小。 Kubernetes 不支援 volume shrinking。

    批次刪除¶

    1. 在儲存列表勾選多個 PVC。
    2. 點批次操作列中的 刪除。
    3. 確認。

    資料會永久消失！ 一次最多 200 個 PVC，請務必通知使用者先備份。

    群組儲存權限設定¶

    群組 PVC 有兩條權限設定路徑：

    flowchart TD
    GroupAdmin["Group Admin\n設定群組 PVC 權限"] -->|批次套用| GroupPVC["Group PVC\n（權限來源）"]
    GroupPVC -->|繼承（唯讀顯示）| ProjectPage["Project Storage 頁面\n（群組成員列）"]
    ProjectAdmin["Project Admin\n管理非群組成員"] -->|直接管理| DirectPerms["Project Storage\n（非群組成員列）"]
    DirectPerms --> ProjectPage

    規則：

    • 群組成員 在綁定到專案的 PVC 上的權限，永遠 由群組決定，無法在專案層級覆寫。
    • 非群組成員 才能在專案層級獨立設定權限。

    Admin 批次設定權限¶

    1. 在 PVC 列點 Permissions。
    2. 勾選多位群組成員。
    3. 設定權限等級：
    權限 意義
    Read 可掛載並讀取
    Write 可掛載、讀取、寫入
    Admin 含權限管理（典型用於群組共同負責人）
    1. 套用。

    JuiceFS 操作流程¶

    允許訓練 job 使用 JuiceFS 路線之前，先跑：

    scripts/storage/verify-juicefs-gpu23.sh
SHARED_RWX_BENCH_STORAGE_CLASS=juicefs-gpu23-rwx scripts/storage/benchmark-shared-rwx.sh

    不要把 production JuiceFS 指到平台內建的單 pod MinIO。 請使用外部分散式 S3-compatible 後端，再用 scripts/storage/bootstrap-juicefs-secret.sh 建立 juicefs-system/juicefs-gpu23-secret。

    容錯測試需涵蓋：

    • 一個 JuiceFS mount pod 重啟
    • 一個 TiKV pod 重啟
    • 一個 worker node 重開

    成功的寫入必須在恢復後仍存在；中斷的訓練 pod 應該能 remount 並 resume，或以 Kubernetes 可預期的失敗結束。
