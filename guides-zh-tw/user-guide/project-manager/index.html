    Project Manager 指南¶

    Project Manager 是專案的「技術負責人」。他可以管理 Config、Build、Image、Storage（非群組成員的部分），但 不能改成員。

    你能做什麼¶

    動作 Project Admin Project Manager Project User
    啟動 Workspace（吃配額） ✓ ✓ ✓
    看 Overview / Plan / Builds / Resources ✓ ✓ ✓
    看 Config 檔案歷史 ✓ ✓ ✓
    編輯 / 上傳 Config ✓ ✓
    觸發 / 取消 Build ✓ ✓
    進 Storage 頁籤、改非群組成員權限 ✓ ✓
    進 Images 頁籤、註冊允許的 image ✓ ✓
    進 Members 頁籤 / 改成員 ✓
    刪除專案 ✓

    看得到的頁籤： Overview / Configurations / Plan / Builds / Resources / Storage / Images。 看不到 Members。

    Configurations（Config 檔案版本管理）¶

    Config 採用 Content-Addressable Storage（CAS）：每個版本都有一個 SHA-256 hash，不可被改寫。

    工作流¶

    1. Projects → [專案] → Configurations 頁籤。
    2. 點 + New Config 上傳 YAML 或在編輯器貼入。
    3. 每次儲存都會建立新的不可變版本。
    4. 點 History 圖示看歷史；點 Deploy 把某版本部署。

    截圖建議： Configurations 頁籤的 file tree、編輯器、版本歷史側欄。

    Builds（容器映像建置）¶

    Builds 只有在平台管理員替專案開啟 Image Build Permission 並綁定有效資源計畫後才能送出。若按鈕是 disabled，請先確認專案治理設定與剩餘 build CPU / memory。

    開始一個 Build¶

    進 Builds 頁籤 → Start Build。三種來源：

    來源 用途
    Upload archive 上傳 .tar.gz 或 .zip（Dockerfile 放在根目錄）
    From Storage 指定一個你有讀權限的 PVC，並輸入子路徑
    Dockerfile only 直接貼入 Dockerfile，平台會生成最小 build context

    Build 設定¶

    欄位 注意
    Image Name 例：trainer-api
    Tag 留空會自動產生時間 + commit；也可以用 v1.0.1
    CPU 核心 不可超過剩餘 build 配額
    Memory（GB） 同上
    Timeout（分鐘） Build job 超過時自動中止

    新版重點： Build job 已支援 CPU / Memory 設定與 Timeout。Build 期間用的資源也會佔用專案配額。

    監看 / 取消¶

    • Build 列表會即時顯示狀態（Pending / Running / Succeeded / Failed）。
    • 點該筆 Build 看 live log（後端使用 live 模式串流）。
    • 你建立的 Build 或你管理的專案 Build 可以從歷史中刪除；如果仍在 Running，後端會先停止 Kubernetes build job，再清理 Harbor artifact 與 allow-list metadata。

    Build 完成的 image 會推到該專案 Group 對應的 Harbor registry path，例如 harbor.cscc.nthu.edu.tw/<group>/<image>:<tag>。實際 registry lane 由群組的 Registry Policy 決定。

    Images（允許的映像清單）¶

    必須是 Manager / Admin 才看得到 Images 頁籤。

    平台啟動 Workspace 時，使用者只能從專案的 允許清單 中選 image。

    操作：

    1. Images 頁籤 → + Add Image。
    2. 輸入完整 image 名稱（例：harbor.cscc.nthu.edu.tw/nlp-lab/trainer:v1.0.1）。
    3. 平台會檢查 image 是否存在、且是否符合該群組的 Registry Policy；不符會被擋下來。
    4. 加入後，所有專案成員在啟動 Workspace 時就能選此 image。

    Storage（非群組成員權限管理）¶

    必須是 Manager / Admin 才看得到 Storage 頁籤。

    工作流：

    1. Storage 頁籤列出該專案綁定的所有 PVC，分成「Group inherited」與「Direct」。
    2. 對於 Direct（非群組成員） 的列，可以新增 / 移除使用者並設定 Read / Write / Admin。
    3. 對於 Group inherited 列，會被標為唯讀，需到對應群組調整。

    截圖建議： 專案 Storage 頁籤，顯示 inherited 列（灰色 / lock icon）與 direct 列。

    申請新 PVC¶

    Manager 不能 直接建立 PVC。請：

    1. 至 Requests → + New Request → 類別 Resource。
    2. 描述：要哪種 storage class、容量、用途、使用者範圍（個人 / Group）。
    3. 平台 Admin 審核 → 在 /admin/storage 建立後，PVC 會出現在你的 Storage 頁籤。

    Plan / Queue（檢視與切換）¶

    Manager 可以看到 Plan 頁籤，包含：

    • 此專案綁定的 Resource Plan
    • 各 Volcano Queue 的優先級、Preemptible、保留 GPU
    • 時段倒數提示：如果計畫的時段（time window）即將關閉，畫面會出現倒數與 切換到 default queue 按鈕

    切到 default queue 的工作不會被時段驅逐，但通常優先級較低、可被搶占。

    截圖建議： Plan 頁籤，顯示 queue 列表、保留 GPU 欄、時段視覺化。

    無法做的事 / 必須轉給 Admin¶

    想做的事 該找誰
    加入 / 移除專案成員 Project Admin
    變更專案 Plan 綁定 平台 Admin（透過 Request）
    建立新的 PVC 平台 Admin（透過 Request）
    把整個 Group 加入專案 Project Admin
    改群組層級的 Storage 權限 Group Admin

    常見問題¶

    我的 Build 一直 Pending？

    通常是專案配額不足（其他 Workspace 把 CPU / Memory 用完）或是該群組沒被允許用對應的 Registry Policy。在 Plan 頁籤確認剩餘配額。

    我看不到 Images / Storage 頁籤？

    請確認你的 Project Role 至少是 manager。Member 看不到這兩個頁籤。

    Build log 一直在轉但沒輸出？

    新版改用 live streaming（live 而非 follow）。如果整個 build pod 卡在 image pull，請看左上角的「stage」標示是否仍在 fetch base image。
