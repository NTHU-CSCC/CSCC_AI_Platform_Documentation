    SonarQube Scanning¶

    This repository is scanned as multiple SonarQube projects so each nested Git repository keeps accurate SCM blame and ownership:

    • CSCC_AI_Platform_Backend
    • CSCC_AI_Platform_Frontend
    • CSCC_AI_Platform_Exporter
    • CSCC_AI_Platform_CSCC_CLI
    • CSCC_AI_Platform_OpenVPN_API
    • CSCC_AI_Platform_InfraDocs

    Local Setup¶

    Copy .env.example to .env and set SONAR_TOKEN. The token must stay local; .env is ignored by Git.

    cp .env.example .env
$EDITOR .env

    Run all scans:

    scripts/ci/sonar-scan.sh

    Run a subset by passing project directories:

    scripts/ci/sonar-scan.sh CSCC_AI_Platform_Backend CSCC_AI_Platform_Frontend

    Select projects from a changed-file list:

    git diff --name-only origin/release...HEAD > /tmp/changed-files.txt
scripts/ci/sonar-select-projects.sh /tmp/changed-files.txt

    GitHub Actions¶

    The root ci.yml workflow runs on self-hosted runners with the self-hosted,platform labels. It detects changed files, maps them to Sonar project directories, prepares coverage for those projects, and runs only the matching Sonar scans. Changes to shared scanner execution or coverage scripts select all projects because they can affect every analysis; workflow, documentation, and lightweight CI helper changes select only the Infra/Docs project.

    The Sonar job checks out the top-level repository first, then initializes only the selected Sonar submodules (CSCC_AI_Platform_Backend, CSCC_AI_Platform_Frontend, and/or CSCC_AI_Platform_Exporter). Frontend-only submodule pointer or file changes therefore run only the frontend scanner. Docs, workflow, and lightweight CI helper changes run only the Infra/Docs scanner; only shared scanner execution or coverage-generation script changes select every Sonar project. When a Go project is selected, the workflow provisions Go 1.25.10 and installs the pinned golangci-lint version required by backend coverage preparation. Backend coverage preparation runs with GOFLAGS=-buildvcs=false because Sonar uses temporary submodule checkouts where VCS stamping metadata can be incomplete. The self-hosted runner must have SSH access to private sibling repositories when one of those submodules is selected; the workflow does not rely on the top-level repository GITHUB_TOKEN to fetch submodules.

    Required repository configuration:

    • Secret SONAR_TOKEN
    • Repository variable or secret SONAR_HOST_URL
    • Optional repository variables SONAR_SCANNER_BIN, SONAR_SCANNER_VERSION, SONAR_SCANNER_SHA256, SONAR_SCANNER_CACHE_DIR, SONAR_SCANNER_JAVA_OPTS, and SONAR_QUALITY_GATE_TIMEOUT

    The scan wrapper first tries SONAR_SCANNER_BIN, then sonar-scanner on PATH. If neither exists, it downloads the pinned SonarScanner CLI version into the runner cache ($RUNNER_TOOL_CACHE/sonar-scanner when available, otherwise $HOME/.cache/sonar-scanner). Downloaded archives are SHA256-verified before execution and installed under a cache lock so concurrent self-hosted jobs do not share partial archives or directories. The default scanner version has built-in checksums for Linux and macOS x64/aarch64 runners. When overriding SONAR_SCANNER_VERSION, also set the matching SONAR_SCANNER_SHA256.

    Before pushing, run the local workflow gate from the root repository:

    scripts/ci/act-push.sh

    Scanner runs wait for the server-side Quality Gate result before returning. The default wait timeout is 600 seconds and can be overridden with SONAR_QUALITY_GATE_TIMEOUT.

    The wrapper runs act push in a temporary local clone so checkout cleanup never touches the real workspace. It tests committed content by default, matching what a real push sends to GitHub. To intentionally include uncommitted local changes, run ACT_PUSH_INCLUDE_WORKTREE=true scripts/ci/act-push.sh; the overlay excludes .git, .env, scanner output, coverage output, virtualenvs, dependency directories, and downloaded SonarQube archives. Exported Sonar environment variables take precedence over .env, and values are passed to act as a temporary secret file. Release secret sync and Pages publishing steps are skipped only when the actor is nektos/act; the validation/build portions still run.

    Coverage Inputs¶

    Generate coverage before scanning when possible:

    • Backend: cd CSCC_AI_Platform_Backend && make test-coverage
    • Frontend: cd CSCC_AI_Platform_Frontend && npm run test:coverage
    • Exporter: cd CSCC_AI_Platform_Exporter/mps-exporter && python -m pytest --cov=. --cov-report=xml:coverage.xml
    • CLI: cd cscc-cli && go test ./... -coverprofile=coverage.out
    • OpenVPN API: cd openvpn-api && go test ./... -coverprofile=coverage.out

    Backend coverage excludes non-runtime scan helpers, Kubernetes manifests, test files, and legacy compatibility no-op adapters (tools/**, k8s/**, deploy/**, **/*_test.go, pkg/logger/logger.go, pkg/k8s/init.go, pkg/k8s/container.go, internal/plugin/builtin/preemption_plugin.go). Owner: backend maintainers. Review trigger: remove an exclusion when the file becomes a runtime behavior surface with meaningful unit-testable branches.

    Frontend Sonar coverage exclusions mirror the Vitest coverage policy in CSCC_AI_Platform_Frontend/vite.config.ts: tests, type declarations, app bootstrap, routing/layout/context/i18n wrappers, UI components, hooks, and page shells are excluded from line coverage while shared utilities and business parsers remain covered. Owner: frontend maintainers. Review trigger: update both Vitest and Sonar together when the coverage policy changes.

    Accepted Risk Rules¶

    The scanner configuration only accepts narrow exceptions:

    • Generated protobuf Python files are excluded from exporter analysis. Rule: *. Pattern: **/*_pb2.py, **/*_pb2_grpc.py. Owner: exporter maintainers. Reason: generated files must be regenerated from protobuf sources instead of hand-edited. Review trigger: generator output is replaced by checked-in handwritten adapters or the protobuf generation path changes.
    • Policy engine fixture manifests under deploy/base/policies/tests/resources/** are excluded from the Infra/Docs project because they intentionally model invalid and pre-mutation workload shapes for admission-policy tests. Owner: platform infrastructure. Review trigger: any fixture is reused as a deployable manifest or moved outside the policy test tree.
    • DaemonSet findings for host namespaces, privileged mode, and explicit Linux capabilities are ignored only for operation-specific DaemonSet manifests. Rules: kubernetes:S6428, kubernetes:S6431, kubernetes:S5849. Pattern: **/*daemonset*.yaml. Owner: platform infrastructure. Reason: GPU, networking, and node-agent DaemonSets require host integration. Review trigger: any DaemonSet privilege, host namespace, or capability changes.
    • kubernetes:S6431 is accepted for deploy/overlays/production/juicefs-storage/minio-direct-disk.yaml because the GPU23 MinIO StatefulSet uses host networking to bind pre-provisioned local disk endpoints. Owner: platform infrastructure. Review trigger: replacement with a routable ClusterIP or storage-network design that removes host networking.
    • iac:S5332 is accepted only for cluster-internal HTTP endpoints in backend ConfigMaps, dashboard-to-backend in-cluster routing, MinIO in-cluster OIDC discovery, and the JuiceFS GPU23 example manifest. Owner: platform infrastructure. Reason: these values target Kubernetes service DNS or example manifests behind platform ingress/proxy boundaries; user-facing traffic remains HTTPS. Review trigger: any endpoint becomes user-facing, crosses namespaces without network policy, or the backing service gains native TLS.
    • iac:S6868 is accepted only for backend pods/exec RBAC in deploy/base/backend/serviceaccount.yaml and the production-sandbox guardrail role. Owner: backend/platform infrastructure. Reason: backend websocket exec and controlled repair flows require Kubernetes exec. Review trigger: websocket exec is removed, split to a narrower ServiceAccount, or Longhorn repair no longer depends on pod exec.
    • iac:S6471 is accepted only for IDE image Dockerfiles under scripts/images/*/Dockerfile. Owner: platform infrastructure. Reason: their shared entrypoint performs root-only user/home setup and then drops privileges with gosu unless PLATFORM_RUN_AS_ROOT=true is explicitly set. Review trigger: entrypoint logic changes, images stop using platform-init.sh, or IDE pods move to mandatory non-root startup.
    • python:S4502 is accepted only for CSCC_AI_Platform_Backend/tools/harbor_sync/webhook_receiver.py. Owner: backend maintainers. Reason: the endpoint is a Harbor machine-to-machine webhook authenticated by WEBHOOK_SECRET; browser session CSRF controls do not apply. Review trigger: cookie/session authentication is added or the endpoint becomes browser-facing.
    • Secret-bearing files, private keys, .env*, generated build output, and dependency directories are excluded from source analysis.

    Temporary maintainability debt exceptions are rule-specific and must not include bugs, vulnerabilities, secrets, or Security Hotspots:

    • Backend rules go:S3776, go:S1192, and go:S107 on **/*.go. Owner: backend maintainers. Reason: legacy service/application/repository code has broad existing complexity, duplicated literals, and long parameter-list debt that is unsafe to rewrite in one scanning bootstrap. Review trigger: remove the exception for any touched package once its local helper/config object refactor lands. Cleanup owner/ticket: backend maintainers, SONAR-DEBT-BACKEND-2026Q3. Deadline: September 30, 2026.
    • Frontend rules typescript:S6759, typescript:S3358, typescript:S7764, typescript:S1874, typescript:S7773, typescript:S7735, typescript:S6571, typescript:S3776, typescript:S4325, typescript:S6479, typescript:S7780, typescript:S6853, typescript:S3863, typescript:S6582, typescript:S4165, typescript:S6594, typescript:S7763, typescript:S6772, typescript:S6481, typescript:S6819, typescript:S6606, typescript:S6749, typescript:S6353, typescript:S6848, typescript:S7744, typescript:S6551, typescript:S7723, typescript:S7741, typescript:S7758, typescript:S7776, typescript:S6754, typescript:S7770, typescript:S6478, typescript:S4123, typescript:S7746, typescript:S5843, typescript:S6767, typescript:S2486, javascript:S7772, and javascript:S7764 on **/*. Owner: frontend maintainers. Reason: legacy React/TypeScript migration debt is large and non-security-critical after the bug/accessibility fixes in this bootstrap. Review trigger: remove per-rule exceptions as readonly props, nested conditionals, deprecated APIs, assertions, and hook complexity are migrated feature by feature. Cleanup owner/ticket: frontend maintainers, SONAR-DEBT-FRONTEND-2026Q3. Deadline: September 30, 2026.

    When a backend package or frontend feature folder is modified for non-trivial work, reviewers must check whether the matching rule suppression can be narrowed or removed for that touched path. New packages and feature folders must not rely on these legacy maintainability suppressions unless this file is updated with a specific accepted-risk entry.

    Any new accepted risk must document the Sonar rule, file pattern, owner, reason, and review trigger in this file before being added to scanner configuration.
