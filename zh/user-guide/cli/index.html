    命令列介面¶

    使用 cscc-cli 進行身份驗證、提交映像建置、串流建置記錄，以及管理您自己的 API 權杖。

    適用對象： 所有已驗證的使用者。

    登入¶

    cscc-cli login --username alice

    CLI 會提示您輸入密碼，並呼叫 /api/v1/cli/login。此流程不使用 CAPTCHA，也不需要 X-API-Key。

    成功後，CLI 會將 cscc_... 使用者 API 權杖以 0600 權限儲存於本機設定檔。後端僅儲存權杖的 SHA-256 雜湊，因此明文權杖只會在建立當下顯示一次。

    可選的權杖名稱：

    cscc-cli login --username alice --name workstation

    TLS / 內部 CA¶

    api.cscc.nthu.edu.tw 可能呈現預設信任儲存區未包含的內部 CA 憑證。 若登入時出現 x509: certificate signed by unknown authority，請設定下列其中一項：

    環境變數 說明
    CSCC_API_URL API base URL，例如 https://api.cscc.nthu.edu.tw
    CSCC_API_CA_CERT_PATH 一個或多個 PEM 檔案，以 , 或 ; 分隔
    CSCC_API_TLS_SERVER_NAME 可選的 TLS SNI 名稱（未設定時退回設定檔）
    SSL_CERT_FILE 包含內部根 CA 的 PEM 檔案路徑
    REQUESTS_CA_BUNDLE 替代環境變數，指向包含內部根 CA 的 PEM 套件

    範例：

    CSCC_API_CA_CERT_PATH=/Users/me/platform-internal-root-ca.pem ./cscc-cli login --username alice

    從設定下載並信任 CLI CA¶

    開啟 https://dashboard.cscc.nthu.edu.tw/settings，並點選 API Key & CLI。

    1. 點選 Download CLI CA，將 platform-internal-root-ca.pem 儲存到您的電腦。
    2. 執行儀表板提供的指令（或手動複製）：
    mkdir -p ~/.config/cscc-cli && curl -fsSL <dashboard-host>/api/v1/me/cli-ca -o ~/.config/cscc-cli/platform-internal-root-ca.pem
    1. 在所有 CLI 指令中使用：
    CSCC_API_CA_CERT_PATH=~/.config/cscc-cli/platform-internal-root-ca.pem cscc-cli whoami

    在具有內部 CA 的機器上首次執行：

    1. 從內部平台管理員的套件中取得 platform-internal-root-ca.pem（或 ca.crt）。
    2. 將其儲存到本機路徑，例如 ~/.config/cscc-cli/platform-internal-root-ca.pem。
    3. 設定 CSCC_API_CA_CERT_PATH 並重試登入。
    4. 若不希望每次都設定環境變數，將該檔案放入 ~/.config/cscc-cli/ 並使用上述其中一個檔名，登入時會自動偵測。

    檢查身份¶

    cscc-cli whoami

    此指令會驗證已儲存的權杖，並列印目前的使用者與角色資訊。角色變更會立即生效，因為授權在每次請求時都會從後端 RBAC/Casbin 狀態解析。

    建置映像¶

    cscc-cli build --project <project-id> --name <image-name> --tag v1.0.1 .

    CLI 會將目標目錄打包為 .tar.gz 建置內容、提交至 /api/v1/images/build，並預設串流建置記錄。該目錄的建置內容根目錄必須包含 Dockerfile。

    實用選項：

    選項 說明
    --project 擁有此映像建置的專案 ID
    --name 簡短的映像名稱，例如 trainer-api
    --tag 可選的映像標籤。省略則使用平台預設值
    --follow=false 提交建置但不串流記錄

    CLI 會從 CSCC_API_URL 或已儲存的登入設定讀取 API 端點。正式環境請使用閘道 API 主機：

    export CSCC_API_URL=https://api.cscc.nthu.edu.tw

    建置請求和記錄串流會使用 cscc-cli login 建立的使用者權杖：

    Authorization: Bearer cscc_...

    CLI 建置不需要 CSCC_API_KEY、PLATFORM_API_KEY 或 X-API-Key 標頭。專案必須啟用 Image Build Permission，且您的專案角色須允許建置。

    權杖管理¶

    列出有效的 API 權杖：

    cscc-cli tokens list

    依 ID 撤銷權杖：

    cscc-cli tokens revoke <token-id>

    從目前機器登出：

    cscc-cli logout

    登出時會先呼叫 DELETE /api/v1/me/api-tokens/current 撤銷目前使用的權杖，然後保留設定中的 TLS 設定並清除本機權杖欄位，讓憑證快取可重複使用。

    預設值¶

    設定 預設值 說明
    USER_API_TOKEN_TTL_HOURS 2160 權杖存活時間（小時，90 天）
    USER_API_TOKEN_MAX_ACTIVE 20 每位使用者最多可有的有效權杖數
    CLI_LOGIN_MAX_ATTEMPTS 5 鎖定前允許的登入失敗次數
    CLI_LOGIN_LOCKOUT_SECONDS 900 鎖定時間（秒）
