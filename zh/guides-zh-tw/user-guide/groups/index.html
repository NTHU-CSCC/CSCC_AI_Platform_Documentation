    Group 與權限¶

    Group 代表一個團隊或實驗室。它讓多個專案共用同一份儲存與一致的策略（Storage Policy / Registry / Root / HostPath）。

    Group 在三角色中的角色¶

    Group 自己有兩種角色（與 Project 三角色不一樣）：

    Group 角色 能做什麼
    Member 取用群組 PVC（依權限）、在被加入的專案中視為成員
    Admin 上述全部 + 新增 / 移除群組成員、批次設定群組 PVC 權限

    你 可以同時 是 Group A 的 Admin、Group B 的 Member、不屬於 Group C，互不影響。

    Groups 列表頁¶

    截圖建議： Groups 列表，顯示 group 卡片（成員數、Storage Policy badge、Registry badge）。

    每張卡片：

    • 群組名稱與描述
    • 成員數
    • Storage Policy / Registry Policy
    • View / Edit / Delete

    進入群組¶

    點群組卡片進入細節頁。頁籤：

    頁籤 顯示什麼
    Members 所有成員 + Group 角色
    Storage 該群組的 PVC 與權限設定

    加入 / 退出群組¶

    加入群組 不能自助。請：

    1. Requests → + New Request → 類別 Access。
    2. Description 寫：「需要加入 Group X，原因是…」。
    3. 等 Group Admin 或平台 Admin 審核。

    退出群組同樣需通知 Group Admin 或平台 Admin。

    新增成員（Group Admin）¶

    需 Group Admin 角色。

    1. 進群組 → Members → + Add Members。
    2. 搜尋 username / email。
    3. 勾選使用者 → Add。

    截圖建議： Add Members 對話框，含 user 多選清單。

    群組儲存權限繼承¶

    群組儲存權限會自動 繼承 到被加入的專案：

    flowchart TD
    GroupAdmin["Group Admin\n設定群組 PVC 權限"] -->|批次套用| GroupPVC["Group PVC（權限來源）"]
    GroupPVC -->|繼承（唯讀）| ProjectStorage["Project Storage 頁面\n的群組成員列"]
    ProjectAdmin["Project Admin / Manager\n管理非群組成員"] -->|直接管理| DirectRows["Project 直接權限\n的非群組成員列"]
    DirectRows --> ProjectStorage

    核心規則：

    • 你是某 Group 的成員 → 你在所有掛該 Group PVC 的專案中，權限永遠由群組決定。
    • 專案層級看到該列會是唯讀，並有 info badge 連回群組設定。
    • 想改？回到群組 Storage 頁籤，用 Group Admin 權限設。

    群組策略（Group Policies）¶

    由平台 Admin 在群組建立 / 編輯時設定：

    策略 影響什麼
    Storage Policy 群組能用哪些 storage class；建立 PVC 時下拉選單會限制
    Registry Policy 群組底下 Project 的 Build 推到哪個 Harbor 路線
    Root 容器允許 群組底下 Project YAML 是否能用 runAsUser: 0，以及 IDE 是否能顯示 root profile
    HostPath 白名單 哪些 hostPath 可以被掛進 pod；其他會被擋下

    一般成員看不到這些設定，但會被它影響：例如選不到某個 storage class、Build 推到指定 registry path。

    常見問題¶

    我看到群組但點不到 + Add Members？

    需 Group Admin 角色。寫信給目前的 Group Admin 升你的角色。

    Project Storage 頁面顯示我的權限是灰的、改不了？

    這是繼承自群組的列。請改去群組的 Storage 頁面，用 Group Admin 改。

    我退出群組了，但仍能看到專案？

    群組內專案的 access 是 群組成員資格 而來的；移除應立即生效。如果還看得到代表退出操作沒完成，請再聯絡平台 Admin。

    為什麼群組的 PVC 不能用某 storage class？

    群組有 Storage Policy 限制，下拉只會顯示被允許的 storage class。要解除限制請通知平台 Admin 在 group 設定中放寬。
