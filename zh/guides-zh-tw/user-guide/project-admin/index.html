    Project Admin 指南¶

    Project Admin 是一個 Project 的「最高負責人」。一個 Project 至少要有一位 Project Admin（通常是建立者）。

    你能做什麼¶

    動作 Project Admin Project Manager Project User
    看 Overview / Plan / Resources / Builds ✓ ✓ ✓
    啟動 Workspace（吃專案配額） ✓ ✓ ✓
    編輯 Config / 部署 ✓ ✓
    看到 Storage 頁籤 + 設權限 ✓ ✓
    看到 Images 頁籤 + 加入 image ✓ ✓
    看到 Members 頁籤 + 改成員 ✓
    把整個 Group 加入專案 ✓
    變更其他成員的 Project Role ✓
    設定其他成員的專案資源配額 ✓
    申請刪除專案 ✓

    唯一只有 Admin 能看到的頁籤是 Members。前端是直接用 useProjectRole().isProjectAdmin 控制。

    進入專案¶

    Projects → 點專案卡片。頁籤列會出現以下八個（有些只有特定角色才看得到）：

    • Overview / Configurations / Plan / Builds / Resources（全部角色都看得到）
    • Storage / Images（Manager 以上才看得到）
    • Members（只有 Admin 看得到）

    管理成員（Project Members）¶

    新增單一使用者¶

    1. 進專案 → Members 頁籤。
    2. 點 + Add Members。
    3. 搜尋使用者名稱 / Email。
    4. 為每位新成員指定 Project Role：
    角色 給誰
    Member / User 一般研究者
    Manager 系統 / 環境負責人
    Admin 共同負責人
    1. 確認新增。

    截圖建議： Members 頁籤的 Add Members 對話框，含 user / group 切換、role 下拉。

    加入整個 Group¶

    1. 同一個 Add Members 對話框切到 Group 分頁。
    2. 選擇要加入的 Group。
    3. 為這群人設一個預設 Project Role（通常是 Member）。
    4. 確認。

    加入 Group 後：

    • 該 Group 的 所有現任 / 未來成員 都自動是這個專案的成員。
    • 群組對 Group PVC 的權限會自動繼承到此專案的 Storage 頁面，且這些列在專案頁面上是 唯讀（要改要回到群組）。
    • 後續從 Group 移除某人，他在此專案的 access 也會自動消失。

    變更 / 移除成員¶

    • 在成員列上的 Role 下拉切換 user / manager / admin。
    • 在成員列上的 Quota 控制設定個別 GPU / CPU / Memory 上限。
    • 點該列的「移除」即時把該人從專案踢出。
    • 勾選多位成員後，可用批次工具列一次移除、改角色或套用相同 quota。

    不可降級自己的角色：前端會擋你把自己從 Admin 降級，避免專案沒有 Admin。 Group 成員不可在專案頁移除：必須回到 Group 成員管理處理，因為權限是從群組繼承。

    把專案綁定到資源計畫（Bind Plan）¶

    雖然「建立 Plan」是平台 Admin 的事，但 將專案綁到既有 Plan 通常需要平台 Admin 操作。Project Admin 可以：

    1. 開 Plan 頁籤確認目前綁定的 Plan、配額、時段。
    2. 若需要更換 Plan，至 Requests 送 Resource Request，描述：
    3. 想換到哪個 Plan
    4. 為什麼（GPU 數量 / 訓練週期）
    5. 平台 Admin 會在 /admin/plans → PlanProjectSelector 完成綁定。

    Storage（含跨群組權限）¶

    Project Admin 可看到 Storage 頁籤，並做：

    • 申請（透過 Request）新的 Group / Project PVC。
    • 為 非群組成員 設定 PVC 權限（Read / Write / Admin）。

    群組成員的權限是繼承的，無法在這裡覆寫。 對應的列會以唯讀顯示，並有一個 info badge 連到該群組的 Storage 頁面。

    刪除專案¶

    1. 先停止所有該專案的 Workspace。
    2. 在專案 Overview → 編輯 → Delete。
    3. 輸入專案名稱確認。

    影響： - Config 版本歷史與部署紀錄永久消失。 - PVC 不會被刪除（保留在儲存頁）。 - 該專案的 Member 紀錄被清除。

    如果是 Group-owned project，可能還需要平台 Admin 先撤銷 group binding 才允許刪除。

    常見場景¶

    有人離開實驗室，要怎麼移除他在所有專案的存取？

    最快的做法是請平台 Admin 把他從對應的 Group 移除。所有以 Group 為單位加入的專案都會自動同步移除。不要 一個一個 Project 手動踢，容易漏。

    我在 Members 頁籤看不到「+ Add Members」按鈕？

    請確認你的 Project Role 真的是 admin。Manager 看不到 Members 頁籤本身。

    我有平台 Admin 權限但看不到某個專案的 Members？

    請在右上角頭像切到 Admin View 模式，平台 Admin 在 Admin View 才會自動視為 Project Admin。
