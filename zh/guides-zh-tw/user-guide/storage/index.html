    Storage 使用¶

    平台所有持久化資料都是 Kubernetes PVC（Persistent Volume Claim），分成兩種：

    Personal Storage Group Storage
    擁有者 你個人 一個 Group
    建立者 平台 Admin（批次初始化） 平台 Admin（依 Request）
    存取者 只有你 群組成員（依個別權限）
    管理頁面 Storage → Personal 頁籤 Storage → Group 頁籤

    Storage 列表頁¶

    截圖建議： Storage 頁面，顯示 Personal / Group 兩個頁籤、容量、狀態、Browse / Permissions 按鈕。

    每筆 PVC：

    • 名稱 / 描述
    • 容量
    • 狀態：Available / Bound / Pending
    • Browse：開啟 FileBrowser
    • Permissions：（只對 Manager / Admin / Group Admin 顯示）

    三角色 RBAC × Storage¶

    下表是 單一 Project 內 三角色看到 Storage 頁籤的差異：

    動作 Project Admin Project Manager Project User
    在 Workspace 啟動表單裡掛載 PVC（依個別權限） ✓ ✓ ✓
    在 Storage 頁面用 FileBrowser 瀏覽（依個別權限） ✓ ✓ ✓
    看到專案的 Storage 頁籤 ✓ ✓
    改 非群組成員 的 PVC 權限 ✓ ✓
    改 群組成員 的 PVC 權限

    群組成員的 PVC 權限永遠由群組決定。 Project 層級看到的列為唯讀，要改要回到對應群組的 Storage 頁籤（需 Group Admin）。

    FileBrowser¶

    點 Browse 開啟瀏覽器：

    • 進 / 出資料夾
    • 上傳檔案（拖放或選檔）
    • 下載檔案 / 資料夾
    • 建立資料夾
    • 刪除（立即生效，沒有資源回收筒）

    在 Workspace 中掛載¶

    在 New Workspace 表單：

    1. Storage 區塊 → + Add Storage。
    2. 從下拉選你能讀的 PVC。
    3. 設 Mount Path（例：/data 或 /workspace/datasets）。
    4. 重複新增其他 PVC。

    建議統一 mount 路徑： 同一個 PVC 在不同 Workspace 用同樣路徑，code 可以寫死路徑而不需條件判斷。

    訓練檔案配置建議¶

    對使用 gpu1 / gpu2 / gpu3 共享資源的訓練：

    路徑 Storage Class 用途
    /datasets JuiceFS group RWX 共享、讀多寫少的訓練資料
    /checkpoints JuiceFS group RWX 持久 checkpoint，可中途 resume
    /scratch 本地 fast RWO job 期間的暫存熱檔

    訓練結束後再把成品從 /scratch 寫回 JuiceFS 路線，可保住 node-local cache 的讀取速度，又避免共享儲存頻繁寫入競爭。

    Storage 權限等級¶

    權限 你能做什麼
    Read 掛載並讀取
    Write 掛載、讀、寫
    Admin 完全控制（包含改其他人的權限）

    申請更多 / 新的 Storage¶

    Project User / Manager 都不能直接建 PVC：

    1. Requests → + New Request。
    2. 類別：Resource。
    3. Description：寫清楚要的 storage class、容量、用途、是否要群組共享。
    4. Submit 後等平台 Admin 審核。

    常見問題¶

    我能看到 PVC 但 Browse 不開？

    沒有至少 Read 權限就不能瀏覽。請通知對應的群組或專案 Admin 加你進來。

    我從 FileBrowser 上傳的檔案，在 Workspace 裡看不到？

    檢查 Workspace 的 Mount Path 是否就是該 PVC，以及檔案放在 mount path 之內。新建立的檔案會即時出現在 mounted filesystem，不需要重啟。

    PVC 一直 Pending？

    通常 1 分鐘內會 bind。超過 5 分鐘還 Pending 請通知平台 Admin（可能 storage class provisioner 卡住）。

    我已被加入某 Group 但 PVC 看不到？

    Group 成員身分要先生效，再看你在該 PVC 的個別權限是否被設定。Group 成員不會自動拿到 Read，要 Group Admin 在 Storage 頁籤批次給。
