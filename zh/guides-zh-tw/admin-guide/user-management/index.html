    新增使用者（User Management）¶

    權限： 僅 Admin 可進入。 路徑：Admin Panel → 使用者（/admin/users）

    使用者列表¶

    進入頁面後可看到所有平台使用者，欄位包含：

    欄位 說明
    Username 登入帳號（LDAP 帳號）
    Display Name 介面顯示名稱
    Email 聯絡信箱
    Role 系統角色（Admin / Manager / User）
    Created 建立日期
    Actions 編輯、刪除

    可用上方的搜尋列依使用者名稱或 Email 過濾，也可用角色下拉選單只顯示某種角色。

    截圖建議： 使用者列表畫面（/admin/users），顯示完整表格、角色 badge 與操作按鈕。

    建立單一使用者¶

    1. 點選右上角 + 新增使用者。
    2. 填寫表單欄位：
    欄位 必填 說明
    Username ✓ 英數字、可含 - / _（與 LDAP 帳號一致）
    Display Name ✓ 使用者全名
    Email ✓ 必須是合法 Email 格式
    Password ✓ 初始密碼，建議要求使用者首次登入後自行更改
    System Role ✓ 預設為 User；視需要選擇 Manager 或 Admin
    1. 點擊 建立使用者。
    2. 使用者立即可用該帳號登入平台。

    小提醒： 平台採用 LDAP 認證；如果你的環境啟用了 LDAP 同步，使用者名稱必須與 LDAP 目錄中的 uid 一致。

    批次建立使用者（Batch Create）¶

    當需要一次匯入大量使用者（例如新學期開課），請使用 Batch Create 流程。

    flowchart LR
    Step1["步驟 1：輸入帳號\n（每行一個 username 或貼上 CSV）"]
    --> Step2["步驟 2：產生密碼 / 預覽\n（顯示即將建立的帳號清單）"]
    --> Step3["步驟 3：確認建立\n（成功 / 失敗結果）"]
    --> CSV["匯出 CSV 報告\n（含初始密碼）"]

    步驟 1 — 輸入帳號¶

    點擊 + 新增使用者 旁邊的 批次建立，在輸入區貼上每行一個的 username：

    s112345
s112346
s112347

    截圖建議： 批次建立 Step 1，顯示 username 輸入區與「產生密碼」按鈕。

    步驟 2 — 預覽與產生密碼¶

    點擊 產生密碼。系統會替每個帳號產生隨機密碼並進入預覽畫面，無效或重複的 username 會顯示紅色警告。

    截圖建議： 批次建立 Step 2，預覽表格顯示 username + 密碼 + 警告。

    步驟 3 — 確認結果¶

    點擊 建立 N 個使用者。完成後表格會標示每筆「成功」或「失敗」，請務必點擊 匯出 CSV 將初始密碼下載保存，因為密碼之後無法再次查看。

    截圖建議： 批次建立 Step 3，結果頁面與 CSV 下載按鈕。

    編輯使用者¶

    1. 在表格列點擊 編輯 圖示。
    2. 修改可編輯欄位（Display Name、Email、Role）。
    3. 若要重設密碼，勾選 重設密碼 並輸入新密碼。
    4. 點擊 儲存。

    角色變更會被審計： 變更使用者角色會在 Audit Logs 產生 role_changed 事件，記錄前後值。

    批次重設密碼¶

    於使用者列表勾選多筆，按右下方批次操作列的 重設密碼，平台會：

    1. 重新產生隨機密碼。
    2. 立即套用到所選帳號。
    3. 自動下載 CSV 給管理員（username,password）。

    重要： 此操作無法復原，且舊密碼立即失效。執行前請先通知使用者。

    刪除使用者¶

    1. 在表格列點擊 刪除 圖示。
    2. 確認對話框中按 刪除。

    刪除後的影響：

    • 使用者帳號被移除，立即無法登入。
    • 該使用者的 執行中 Workspace 不會自動關閉，請先停止。
    • 專案成員資格會一併移除。
    • 個人 PVC 資料保留，需另行至儲存頁面清理。
    • 過往的 Audit Log 仍會保留並引用該使用者 ID。

    系統角色對照¶

    角色 等級 典型使用情境
    User 2 一般研究者 / 學生
    Manager 1 系所助教 / 部門協調員（可閱覽 Admin Dashboard、Audit Log）
    Admin 0 平台維運人員

    更細的權限請看 RBAC Management（英文版）。
