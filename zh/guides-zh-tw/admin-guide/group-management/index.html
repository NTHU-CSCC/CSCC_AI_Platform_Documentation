    新增群組（Group Management）¶

    群組（Group）代表一個團隊、研究室或部門。群組可以：

    • 擁有共享儲存空間（Group PVC）。
    • 被加入多個專案做為成員（同一份 Storage 政策可重複利用）。
    • 套用群組層級的「儲存策略」、「Registry 策略」、「HostPath 白名單」、「Root 容器允許」。

    權限： 僅 Admin 可建立 / 刪除群組。Manager 不可建立群組。 路徑：Admin Panel → 群組（/admin/groups）。

    群組列表頁¶

    截圖建議： /admin/groups 完整表格，顯示群組名稱、描述、成員數、儲存策略 badge、Registry 策略 badge、編輯 / 刪除按鈕。

    每一列會顯示：

    欄位 說明
    群組名稱 / 描述 由建立者輸入
    成員數 該群組目前的人數
    Storage Policy 例：預設儲存、juicefs-gpu23-rwx
    Registry Policy 例：預設 Harbor、自訂 Harbor 路線
    Actions 編輯、刪除

    建立群組¶

    1. 點選右上角 + 建立群組。
    2. 填寫基本資料：
    欄位 說明 範例
    名稱 英數字、可含 -，會用於 Kubernetes label / namespace 識別 nlp-lab
    描述（選填） 群組用途 自然語言處理研究室
    1. 設定 群組策略（這是新版功能，會直接限制群組底下專案能怎麼使用儲存與 Registry）：
    欄位 說明
    Storage Policy（儲存策略） 此群組新建立的 PVC 會套用的 Longhorn / JuiceFS 儲存類別。預設為平台共享 longhorn pool。
    Registry Policy（Registry 路線） 此群組的 image build 會推送到的 Harbor registry 路線。預設為平台 Harbor。
    Root 容器允許 是否允許此群組擁有的專案在 YAML 中明確設定 runAsUser: 0 或 runAsNonRoot: false。預設關閉。
    HostPath 白名單 此群組的專案只能掛載這裡列出的 hostPath；可指定 read-only / read-write。
    1. 點擊 建立。

    建立者自動成為 Group Admin。 若要讓平台管理員以外的人能管理群組，請把他加為群組成員並把角色設為 admin。

    群組策略詳細說明¶

    Storage Policy（儲存策略）¶

    此選項會限制此群組之後建立的 Group PVC、以及群組下專案掛載的儲存類別。常見選項：

    Storage Class 用途
    longhorn 預設、單節點高 IOPS、適合個人或小型專案
    juicefs-gpu23-rwx 多節點共享 RWX，適合訓練資料集 / checkpoints
    longhorn-gpu23-group GPU23 /data2 舊版 Longhorn 群組儲存
    longhorn-gpu23-rwx-25g 舊 NFS 相容備援

    訓練工作負載建議：資料集與 checkpoint 放 juicefs-gpu23-rwx；單一 job 暫存檔 放本地 RWO，最後再寫回 JuiceFS。

    Registry Policy¶

    選擇此群組的 Project Build 應該推到哪個 Harbor 路線。對應到 Harbor 中的 robot account 與 project namespace。一般使用者只會在 Dashboard 的 Builds / Images 頁面看到結果；Harbor UI 本身是管理服務，預設保留給 admin 使用。

    常見路線：

    Registry Policy 用途
    default 推送到 harbor.cscc.nthu.edu.tw 的平台預設 Harbor lane
    gpu23-harbor 推送到 GPU23 專用 Harbor lane；此群組的 image build 會進入 platform-build-gpu23，排到 gpu2/gpu3，CPU/RAM 仍受限，disk 使用 GPU23 /data2 scratch lane（每個 build 200Gi，namespace 總量 2Ti）

    Root 容器允許¶

    關閉時，此群組底下的專案 YAML 會被強制套用 runAsNonRoot: true，並在嘗試以 root 啟動時被 admission controller 拒絕。

    HostPath 白名單¶

    每筆規則包含「Host 來源路徑」與「權限（唯讀 / 讀寫）」。只有列在這裡的路徑 才能透過 configfile HostPathMountPolicy 或 Kubernetes hostPath volume 掛載；其他路徑會被擋下。

    管理群組成員¶

    1. 在群組列表點擊群組名稱進入細節頁。
    2. 切到 成員（Members） 頁籤。

    3. 點 + 新增成員：

    4. 在彈出視窗搜尋 username 或 Email。

    5. 勾選一或多位使用者。
    6. 點 新增 完成。

    截圖建議： 群組成員頁的「新增成員」對話框。

    變更成員角色¶

    群組角色 能做什麼
    Member 取用群組儲存（依 PVC 權限）、被視為群組成員加入專案
    Admin 上述全部 + 新增 / 移除成員、調整群組 PVC 權限

    操作：在成員列表使用 Role 下拉切換 Member / Admin。

    移除成員¶

    點該列的 Remove 圖示，確認後立即生效。

    注意： 從群組移除某使用者，不會 自動撤銷他在群組 PVC 上的個別權限。請到群組的「Storage」頁籤檢查並手動撤銷。

    刪除群組¶

    1. 於群組列表點 刪除。
    2. 確認對話框。

    資料並不會被刪除： 群組所有的 PVC 仍會留在 Kubernetes 上。請先在 儲存管理 中清理孤兒 PVC，再刪除群組。

    把群組加入專案¶

    要讓整個群組成為某專案的成員（讓所有群組成員一次都能進該專案），請：

    1. 進入該專案頁 → Members 頁籤（需 Project Admin 權限）。
    2. 點 + Add Members。
    3. 切到 Group 分頁、選擇要加入的群組、給定群組成員預設角色（Member / Manager / Admin）。
    4. 確認。

    加入後，該群組的儲存權限會自動繼承到專案的 Storage 頁面，且為唯讀（無法在專案層覆蓋；要改權限請回到群組的 Storage 頁籤調整）。
