    Runbook: migrate the cluster CNI from Calico to Cilium (keep Multus)¶

    Rolling, per-node migration of the live kubeadm HA cluster (gpu1/gpu2/gpu3, K8s v1.35.0) from Calico v3.27.0 to Cilium — native routing, eBPF kube-proxy replacement — while Multus and its macvlan/Whereabouts secondary networks keep working. One node at a time, with an acceptance gate between each. Driver: scripts/setup/15-migrate-cni-to-cilium.sh.

    Fresh rebuilds already install Cilium via scripts/setup/05-restore-platform.sh (install_cilium). This runbook is only for converting the existing Calico cluster in place.

    Why these guardrails (do not skip)¶

    • cni.exclusive=false — Cilium's agent otherwise deletes every other file in /etc/cni/net.d, wiping the Multus 00-multus.conf and the lh25g0 macvlan config and breaking Longhorn's 25G storage network.
    • Pin Cilium to the 10G fabric (HA_CILIUM_DEVICES = the iface owning 192.168.110.10X). Cilium must never bind/masquerade the 192.168.109.x access NIC (ingress, OpenVPN, ArgoCD, SSH — the lifeline) or the 25G storage NIC. After every node change we re-probe https://192.168.109.201/.
    • New pod CIDR 10.245.0.0/16 (cluster-pool) so it can't collide with Calico's 10.244.0.0/16 during the dual-CNI window.
    • Order: gpu3 → gpu2 → gpu1 (primary control plane last). Drain fully before each node.

    Prerequisites¶

    • kubectl, helm, curl, cilium CLI on the admin host; kubeconfig at HA_KUBECONFIG (default /tmp/cscc-ha-control-plane/admin.conf).
    • SSH+sudo to all nodes: the scripts read SUDO_PASSWORD from the repo .env (gitignored) into SSHPASS. Never echo or commit it.
    • Set the 10G device pin: export HA_CILIUM_DEVICES=<10G-iface> (discover it in preflight; it may differ per node — use a glob like enp1+ or set per run).

    Procedure¶

    cd /home/user/CSCC_AI_Platform

# 0) Pre-flight (read-only): nodes, NADs, kube-proxy, ingress, the 10G iface per node.
scripts/setup/15-migrate-cni-to-cilium.sh preflight

# 1) Install Cilium DORMANT — agents run but write no CNI conf; kube-proxy stays in charge.
scripts/setup/15-migrate-cni-to-cilium.sh install-dormant
#    Verify: cilium pods Running, NO 05-cilium.conflist on any node, Calico/Multus/workloads healthy.

# 2) Migrate one node at a time. Each call: cordon→drain→label→restart cilium→reboot→
#    remove Calico conf→restart Multus→uncordon→ACCEPTANCE GATE.
MIGRATE_CONFIRM=migrate-gpu3 scripts/setup/15-migrate-cni-to-cilium.sh migrate-node gpu3
MIGRATE_CONFIRM=migrate-gpu2 scripts/setup/15-migrate-cni-to-cilium.sh migrate-node gpu2
MIGRATE_CONFIRM=migrate-gpu1 scripts/setup/15-migrate-cni-to-cilium.sh migrate-node gpu1

    Acceptance gate (must pass before the next node): - new test pod on the node gets a 10.245.x IP; pod↔pod across migrated+unmigrated nodes works; - ClusterIP Service + CoreDNS resolve; - ingress https://192.168.109.201/ and SSH still reachable; - cscc-storage macvlan still attaches — re-run LH_VERIFY_STORAGE_NETWORK=true scripts/setup/11-install-longhorn-storage-network.sh verify, or check a Longhorn pod's k8s.v1.cni.cncf.io/network-status.

    If a gate fails → roll back that node (below) and stop.

    # 3) Make Cilium the cluster default and DELETE Calico. Point of no easy return —
#    only after all three nodes pass + soak.
MIGRATE_CONFIRM=finalize-remove-calico scripts/setup/15-migrate-cni-to-cilium.sh finalize

# 4) Stage B — replace kube-proxy with Cilium eBPF, then remove kube-proxy.
#    Highest 109.x/ingress risk: ingress is re-checked after the rollout and after each node cleanup.
MIGRATE_CONFIRM=kube-proxy-free scripts/setup/15-migrate-cni-to-cilium.sh kube-proxy-free

# 5) Verify
scripts/setup/15-migrate-cni-to-cilium.sh verify
cilium status --wait && cilium connectivity test

    Multus reconciliation¶

    Multus (thick) picks its default delegate as the lexically-first conf in /etc/cni/net.d. Cilium writes 05-cilium.conflist; Calico's was 10-calico.conflist. Step 6 of migrate-node removes the Calico conf and restarts the Multus daemon on the node, so the default delegate becomes Cilium with no Multus DaemonSet change. The macvlan/Whereabouts NAD is an independent delegate — unaffected.

    Troubleshooting¶

    cilium-envoy CrashLoopBackOff — "Too many open files"¶

    Symptom: every cilium-envoy pod loops; logs show evutil_make_internal_pipe_: pipe: Too many open files / evsig_init_: socketpair: Too many open files at startup. Cause: these GPU nodes have very high core counts, envoy spawns one worker per core, and the container inherits containerd's soft LimitNOFILE=1024 (no default_ulimits set) — so envoy exhausts its fd budget before it finishes booting. Node-wide limits are fine (fs.inotify, fs.file-max); it is strictly a per-process cap.

    Fix (rolling, one node at a time; containerd restart keeps running containers alive via their shims):

    ssh <node> 'sudo install -D -m0644 /dev/stdin /etc/systemd/system/containerd.service.d/10-nofile.conf <<EOF
[Service]
LimitNOFILE=1048576
EOF
sudo systemctl daemon-reload && sudo systemctl restart containerd'
# then recycle envoy on that node so the new container inherits the limit:
kubectl -n kube-system delete pod -l k8s-app=cilium-envoy --field-selector spec.nodeName=<node>

    Gate after each node: cilium-envoy 1/1 0 restarts, node Ready, ingress 192.168.109.201 still answers. Order gpu3 → gpu2 → gpu1 (gpu1 last = primary control plane).

    Rollback¶

    • Per node (Stage A, before finalize): Calico is still installed and authoritative on un-migrated nodes, and cni.uninstall=false, so one node reverts cleanly: 
      kubectl label node <node> io.cilium.migration/cilium-default-
ssh <node> 'sudo kubeadm ... ' # restore 10-calico.conflist (re-apply Calico DS triggers it)
kubectl -n kube-system delete pod -l name=multus --field-selector spec.nodeName=<node>
kubectl uncordon <node>   # after reboot
    • Whole migration before finalize: helm uninstall -n kube-system cilium; re-label/reboot any migrated node back to Calico.
    • After finalize (Calico deleted): costlier — re-run install_calico from 05-restore-platform.sh and reverse labels. So do not run finalize until all three nodes pass their gates and soak.

    Key config (overridable env; defaults in scripts/setup/lib/common.sh)¶

    Var Default Meaning
    HA_CILIUM_VERSION 1.18.1 verify K8s 1.35 in Cilium support matrix before bumping
    HA_CILIUM_POD_CIDR 10.245.0.0/16 new cluster-pool CIDR
    HA_CILIUM_DEVICES (autodetect) pin to the 10G iface; keeps Cilium off 109.x/25G
    HA_K8S_SERVICE_HOST/PORT 192.168.110.200 / 6443 API VIP for kube-proxy-free mode
    HA_INGRESS_VIP 192.168.109.201 109.x lifeline probed after every node change
